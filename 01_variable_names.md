# Variable names
- имена должны передавать намерения программиста;
- хорошее имя описывает проектную сущность, решение;
- имена должны быть максимально конкретны.

Пример:
```
//before - after

DEF_CAPACITY - DEFAULT_CAPACITY              // константное значение по умолчанию для емкости динамического массива

slots - keys                                 // массива для хранения ключей в словаре

count - size                                 // размер(колличество элементов) динамического массива

count - rotationCount                        // колличество элементов на которое нужно повернуть очередь      

expressionStack - arithmeticElements         // коллекция элементов арифметического выражения

element - arithmeticElement                  // элемент арифметического выражения

result - arithmeticResult                    // результат арифметичекского вычесления

sum - nodesValuesSum                         // сумма значений узлов связанного списка

temp - referenceIndex                        // индекс с которого начинается циклический проход по массиву, для сравнения с текущим индексом

index - currenIndex                          // меняющийся индекс для прохода по массиву

index - cachedSlotIndex                      // индекс слота кешированных эелемнтов структуре данных кэша

index - emptySlotIndex                       // индекс пустого слота в структуре данных кэша

indexOfMinHits - indexOfSlotWithMinimalHint  // индекс слота в кэше с минимальным колличеством обращений  
```
